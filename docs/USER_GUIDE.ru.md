# Jumix — руководство для платформенного администратора

> Это руководство для **суперадминистратора платформы** (роль `superadmin`) — сотрудника ТОО «Telse», который одобряет организации, крановщиков и технику.
> Роли `owner` (владелец организации) и `operator` (крановщик) имеют свои интерфейсы и здесь не рассматриваются — у них отдельные guides.

---

## 1. Первый вход

После развёртывания платформы ИП-исполнитель создаёт вам superadmin-аккаунт и передаёт телефон + временный пароль.

1. Откройте `https://jumix.kz` (или адрес вашего сервера)
2. На странице входа введите:
   - **Телефон:** `+7 XXX XXX XX XX` (в формате Казахстана)
   - **Пароль:** временный пароль от исполнителя
3. Нажмите «Войти»

При первом входе **смените пароль** через настройки аккаунта (в верхнем правом меню).

> Если забыли пароль — воспользуйтесь восстановлением по SMS. На ваш телефон придёт код, после ввода сможете задать новый пароль.

---

## 2. Обзор платформы (Dashboard)

После входа вы попадаете на `Обзор платформы` — главная страница. Здесь видно:

- **Метрики pending-очередей:** сколько заявок ждут вашего рассмотрения (крановщики / трудоустройства / краны)
- **Лента audit-событий:** последние действия на платформе (кто что создал, одобрил, изменил)
- **Сводка организаций:** последние 5 зарегистрированных компаний

Клик на карточку метрики → переход на соответствующую очередь.

## 3. Одобрение заявок (главная задача)

Раздел **«Заявки на рассмотрение»** (`/approvals`) — ключевая рабочая область superadmin'а.

Три вкладки:

### 3.1. Крановщики

Каждый новый крановщик, зарегистрировавшийся через мобильное приложение, попадает сюда.

**Что проверять:**
- ИИН (должен быть валидным)
- ФИО и аватар
- Наличие удостоверения крановщика
- Срок действия удостоверения (не истёк)

**Действия:**
- **Одобрить** — крановщик получает статус `approved` и может быть нанят организациями
- **Отклонить** с причиной — укажите почему (например, «удостоверение просрочено», «ИИН не подтверждается»). Причина видна крановщику в его мобильном приложении

### 3.2. Трудоустройства (hires)

Владелец организации подал заявку на найм крановщика — она ждёт вашего одобрения (инвариант холдинг-модели: внешний актор обязателен).

**Что проверять:**
- Крановщик не в конфликте интересов (не работает у нескольких конкурирующих организаций)
- У организации хороший рейтинг (для MVP — формально, пока без автоматизации)

**Действия:** Одобрить / Отклонить с причиной.

### 3.3. Краны

Новая единица техники от организации.

**Что проверять:**
- Инвентарный номер не дубликат
- Разумные параметры (грузоподъёмность, год выпуска)

**Действия:** Одобрить / Отклонить.

## 4. Управление организациями

Раздел **«Организации»** (`/organizations`).

### 4.1. Создать новую организацию

1. Нажмите «Новая организация»
2. Заполните:
   - **Название:** полное название (ТОО/ИП + юридическое имя)
   - **БИН:** 12 цифр, проверяется checksum'ом
   - **Контакты:** имя / телефон / email
   - **Телефон владельца:** на него придёт SMS-инвайт
3. Нажмите «Создать»

Владелец получит SMS с инструкцией задать пароль, после чего сможет войти в свой кабинет.

### 4.2. Приостановить / возобновить / архивировать

Открыв карточку организации, вы видите 3 действия:

- **Редактировать** — изменить название, контакты, БИН
- **Приостановить** → организация недоступна владельцу до возобновления. Используется при временной приостановке (нарушения, спор)
- **Активировать** → вернуть из приостановленного статуса
- **Архивировать** → **необратимое действие**. Организация полностью заблокирована. Используется для банкротства, закрытия

> Архивация — терминальный статус. Восстановить архивированную организацию нельзя (только пересоздать как новую).

## 5. Мониторинг

### 5.1. Журнал событий

На dashboard'е в правой колонке — лента последних 20 audit-событий. Полный журнал — пока через БД (MVP).

Если видите подозрительные события (множественные попытки логина, массовое одобрение/отклонение) — свяжитесь с исполнителем.

### 5.2. Глобальные списки

- `/crane-profiles` — все крановщики платформы (поиск по ФИО / ИИН)
- `/cranes` — весь парк техники всех организаций
- `/organization-operators` — все активные и исторические назначения

Эти страницы для аналитики и поиска — массовые действия отсюда недоступны.

## 6. Что делать, если...

### 6.1. Владелец не может войти

1. Попросите его проверить телефон (формат +7 XXX XXX XX XX)
2. Предложите восстановление по SMS (кнопка «Забыли пароль» на login)
3. Если SMS не приходят — см. раздел «6.4. SMS не приходят»
4. Крайняя мера: суперадмин сбрасывает пароль через БД (обратиться к исполнителю)

### 6.2. Крановщик жалуется, что заявка отклонена без причины

Откройте карточку крановщика в `/crane-profiles` → найдите `rejectionReason` в истории. Если причина не была указана при отклонении — это баг, сообщите исполнителю.

### 6.3. Организация не видит своих кранов / сотрудников

Скорее всего, она `suspended` или `archived`. Проверьте статус в `/organizations`. Активируйте при необходимости.

### 6.4. SMS не приходят

- Проверьте баланс на счёте Mobizon (SMS-шлюз)
- Попросите пользователя попробовать ещё раз через 1 минуту (rate limit — 1 SMS / 60 сек на номер)
- Если не работает — свяжитесь с исполнителем

### 6.5. Сайт не открывается

Проверьте:
- `https://jumix.kz/health` — должен вернуть `{"status":"ok"}`
- Если health не отвечает — свяжитесь с исполнителем (проблема на сервере)

### 6.6. Я ошибся с одобрением / отклонением

- Для крановщика: можно повторно отклонить уже одобренного (статус меняется), но массово **не пропуская** — создаёт audit-шум
- Для организации: suspended ↔ active переключается свободно
- Для уволенного крановщика: восстановить нельзя, нужна **новая заявка на найм** от владельца (инвариант state machine)

## 7. Безопасность

- **Не делитесь паролем superadmin** — это учётка с максимальными правами
- При подозрении на компрометацию — немедленно смените пароль и свяжитесь с исполнителем
- Все действия superadmin'а записываются в audit log (навсегда)
- Логаут на чужом устройстве — через «Выйти» в правом верхнем меню
