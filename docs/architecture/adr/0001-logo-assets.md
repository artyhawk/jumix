# ADR-0001: Brand assets — PNG-only на MVP

- **Дата:** 2026-04-21
- **Статус:** Accepted
- **Автор:** Yerbol
- **Контекст:** `apps/web/public/brand/`

## Контекст

Дизайнер предоставил только PNG-версии логотипа (без SVG):
- `logo-full.png` — 5504×1642, полный логотип (знак + текст), цветной, прозрачный фон
- `logo-mark.png` — 1650×1642, знак "J" без текста, цветной, прозрачный фон

Монохромной версии (white-on-transparent) нет.

Формат PNG не подходит для:
- favicon разных размеров (16/32/180) без потери качества
- retina-дисплеев без экспорта 2x/3x вариантов
- изменения цвета под тему (монохромные плашки)
- email-шаблонов которые часто фильтруют большие изображения

## Решение

**На MVP используем PNG напрямую**, с компрессией и ресайзом через `next/image`. Для favicon сгенерируем набор размеров через `sharp` при сборке — качество 1650×1642 → 16×16 приемлемое.

**Brand-палитру зафиксировали** по извлечённому HEX из `logo-mark.png`:
- База: HSL 27.5° / 95% / 52% → `#F97B10` (`brand-500`)
- Шкала пересчитана через варьирование L (см. `CLAUDE.md §8.2`)

Воспроизведение:
```bash
python3 -c "
from PIL import Image
img = Image.open('apps/web/public/brand/logo-mark.png').convert('RGBA')
# фильтр оранжевых пикселей, медианный тон — см. git log этого ADR
"
```

## Последствия

**Положительные:**
- Не блокирует запуск проекта.
- Brand-палитра формально привязана к реальному HEX лого (не subjective).

**Отрицательные:**
- favicon на high-DPI дисплеях будет слегка мягче, чем с SVG.
- Добавление монохромной вариации (например, для печати или тёмных хедеров) потребует Photoshop/Figma-ручной работы до получения SVG.
